# ADR-009: 하루 단위 정리 흐름을 기본 분석 단위로 삼는다

## 상태

Accepted

## 결정 시점

2026-01-04

## 배경

ProblemWarruru에서 로그를 어떻게 묶어 분석할지에 대한 기준이 필요했다.

- 세션 단위만으로는 맥락이 단절될 수 있다
- 실제 개발 흐름은 하루 단위로 회고되는 경우가 많다
- 자동 묶음 기준은 의도치 않은 결과를 만들 수 있다

시간 단위, 세션 단위, 자동 클러스터링 등이 선택지로 존재했다.

## 결정

ProblemWarruru의 기본 정리 단위는 하루로 설정한다.

하루 동안 생성된 세션의 요약본을 사람이 직접 확인하고,
정리 및 글 생성 여부를 선택한다.
요약의 경우 AI를 통해 진행한다.

## 이유

- 개발 회고 흐름과 자연스럽게 맞는다
- 자동 판단을 배제할 수 있다
- 기록을 통제 가능한 단위로 유지할 수 있다

의도적으로 포기한 부분은 다음과 같다.

- 자동 세션 묶음
- 주제 기반 자동 분류
- 시간 외 기준의 집계

## 결과

- 하루 단위 세션 요약 개념이 도입되었다
- ProblemWarruru UI 설계의 기준이 마련되었다
- 블로그 초안 생성 흐름이 명확해졌다

이 기준은 이후 데이터 모델과 UI 설계에 영향을 준다.
