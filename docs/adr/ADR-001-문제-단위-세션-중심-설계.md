# ADR-001: 문제 단위(Session) 중심 설계

## 상태
Accepted

## 배경
일반적인 채팅 서비스는 대화를 시간순 메시지 묶음으로 취급한다.
그러나 와르르랩의 목적은 단순한 대화 기록이 아니라,
하나의 문제를 해결하기까지의 사고 과정을 남기는 것이다.

이를 위해 "무엇을 기준으로 기록을 묶을 것인가"에 대한 결정이 필요했다.

## 결정
하나의 문제를 하나의 Session으로 정의한다.

Session은 로그인 세션이나 HTTP 세션이 아니라,
하나의 문제를 해결하기 위한 사고 단위이다.

## 이유
- 하나의 Session은 하나의 트러블슈팅 글 단위로 이어질 수 있다.
- 문제별로 시도, 실패, 해결을 명확히 구분할 수 있다.
- 나중에 ProblemWarruru에서 분석하기에 가장 적합한 단위이다.

## 결과
- 모든 메시지는 반드시 Session에 소속된다.
- Session은 OPEN / RESOLVED 상태를 가진다.
