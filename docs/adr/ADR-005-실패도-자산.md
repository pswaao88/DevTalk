# ADR-005: 실패도 기록 자산으로 취급한다

## 상태
Accepted

## 배경
일반적인 개발 기록에서는 실패 로그를 제거하거나 숨긴다.
그러나 실제로 문제를 해결하는 데 가장 중요한 정보는
"왜 실패했는가"에 있다.

## 결정
실패를 정상적인 기록 데이터로 취급한다.

- 타임아웃
- 429 (Rate Limit)
- 네트워크 오류
- API 호출 실패

위 모든 경우를 Message로 기록한다.

## 이유
- 실패 원인은 나중에 가장 중요한 분석 재료가 된다.
- ProblemWarruru에서 실패 원인을 정리하는 데 필요하다.
- 성공만 남기면 사고 과정이 왜곡된다.

## 결과
- Message.status에 FAILURE를 포함한다.
- 실패 메시지도 세션 타임라인에 그대로 노출한다.
