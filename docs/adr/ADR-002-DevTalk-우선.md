# ADR-002: 개발톡(DevTalk) 우선 개발 전략

## 상태
Accepted

## 배경
와르르랩의 첫 번째 목표는 "기록을 남기는 것"이다.
기록을 남기기 위해서는 사용자가 실제로 매일 사용할 수 있는 도구가 필요하다.

## 결정
DevTalk의 목표를 다음과 같이 설정한다.

- 실제로 매일 사용할 수 있을 것
- 코드와 로그를 편하게 붙여넣을 수 있을 것
- 불편하지 않아 계속 쓰게 될 것

## 이유
- 채팅 웹이 불편하면 로그는 쌓이지 않는다.
- 로그 분석이나 블로그 생성은 기록이 쌓인 이후에 의미를 가진다.
- 기능보다 사용성이 우선이다.

## 결과
- 2회차 개발 목표를 "채팅 MVP 완성"으로 설정했다.
- 블로그 생성 기능은 의도적으로 범위에서 제외했다.
