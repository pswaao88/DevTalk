# ADR-008: Resolved는 자동 처리 대상이 아니라 사람의 판단 선언이다

## 상태

Accepted

## 결정 시점

2026-01-03

## 배경

Resolved 상태를 어떻게 사용할지에 대한 정의가 불분명할 경우,
자동화된 판단이나 잘못된 글 생성으로 이어질 가능성이 있었다.

- AI가 해결 여부를 판단하도록 확장될 여지가 있었다
- 실패 분석 기록과의 관계도 명확히 할 필요가 있었다

Resolved의 역할을 기술적 상태로 둘 것인지,
의미적 판단 기준으로 둘 것인지가 고민되었다.

## 결정

Resolved는 사람이 직접 판단하여 선언하는 상태로 정의한다.

Resolved는 다음을 의미한다.

- 문제 해결에 대한 판단이 끝났음
- 글의 결론으로 사용할 수 있는 해결이 확정되었음

## 이유

- 자동 판단은 프로젝트 철학과 맞지 않는다
- 실패 시도 또한 중요한 기록 자산이다
- 판단의 주체를 명확히 사람으로 고정하기 위함이다

의도적으로 포기한 부분은 다음과 같다.

- 자동 Resolved 추천
- 해결 확률 기반 판단
- AI에 의한 상태 전이

## 결과

- Resolved는 수동 API로만 변경된다
- 실패 분석 글은 Resolved 없이도 가능하다
- ProblemWarruru 설계의 기준점이 명확해졌다

이 정의는 이후 글 생성 로직과 UI 설계에 직접적인 영향을 준다.
