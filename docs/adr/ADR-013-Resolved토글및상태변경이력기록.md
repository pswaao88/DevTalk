# ADR-014-Resolved토글및상태변경이력기록결정

## 상태
Accepted

## 결정 시점
2026-01-06

## 배경
Resolved를 문제 해결의 최종 상태로 고정할 경우,  
“해결된 줄 알았으나 재발한 문제”를 자연스럽게 기록하기 어렵다.

또한 Resolved 이후 채팅을 제한하면 사고 흐름이 단절되고,  
기록 프로젝트의 목적과 어긋난다.

## 결정
- Resolved / Active 상태 전환을 허용한다.
- 상태 전환은 잠금이 아닌 판단 시점 표시로 사용한다.
- 모든 상태 변경은 로그로 기록한다.

## 이유
- 현실적인 개발 흐름을 반영하기 위함이다.
- 판단이 바뀌는 과정 자체를 기록 자산으로 남기기 위함이다.
- 자동 판단과 명확히 선을 긋기 위함이다.

## 결과
- Session은 현재 판단 상태만 유지한다.
- 판단 변화 이력은 로그 타임라인에서 확인 가능하다.
- 이후 자동 정리 및 블로그 생성 단계로 자연스럽게 확장할 수 있다.
