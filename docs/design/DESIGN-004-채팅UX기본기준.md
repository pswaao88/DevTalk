# DESIGN-004-채팅UX기본기준

## 목적
DevTalk에서 기록 중심의 채팅 흐름을 유지하기 위해,  
최소한으로 보장되어야 할 채팅 UX 기준을 정의한다.

본 문서는 UI 구현이 아니라,  
로그 흐름이 깨지지 않도록 하기 위한 사용성 기준을 정리하는 것을 목표로 한다.

## 내용

### 기본 표시 기준
- 하나의 Session 화면에서 해당 Session의 모든 Message를 확인할 수 있어야 한다.
- Message는 시간 순서대로 표시되어 대화 및 판단 흐름이 자연스럽게 이어져야 한다.
- MessageRole(USER / AI / SYSTEM)과 MessageStatus는 구분 가능해야 한다.

### 응답 대기 및 실패 처리
- AI 응답 대기 중임을 사용자가 인지할 수 있어야 한다.
- LLM 응답 실패가 발생하더라도 채팅 흐름은 중단되지 않아야 한다.
- FAILED 상태의 메시지도 동일한 흐름 안에서 표시되어야 한다.

### 요약의 위치
- 요약은 선택적으로 제공될 수 있으나, 로그를 대체하지 않는다.
- 요약을 보지 않더라도 로그만으로 문제 해결 과정을 이해할 수 있어야 한다.

## 결과
위 기준을 통해 다음을 보장한다.

- 기록으로서의 대화 흐름이 UI로 인해 왜곡되지 않는다.
- 실패 이후에도 사고 흐름을 유지할 수 있다.
