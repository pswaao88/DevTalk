# DESIGN-005-하루단위세션흐름정의

## 목적
DevTalk에서 생성된 여러 Session을 하루 단위로 묶어 정리하고,  
문제 해결 기록을 자산화하기 위한 흐름을 정의한다.

본 문서는 자동 요약이나 자동 발행이 아닌,  
사람의 선택을 중심으로 한 정리 흐름을 전제로 한다.

## 내용

### 하루 단위 묶음 개념
- 하나의 날짜에는 여러 개의 Session이 생성될 수 있다.
- 하루 단위 묶음은 시간 기준의 그룹이며, 새로운 도메인 객체는 아니다.
- 각 Session은 독립적인 사고 흐름 단위를 유지한다.

### 세션 상태 표시
- 하루 단위 조회 시 각 Session의 현재 상태를 함께 확인할 수 있어야 한다.
    - ACTIVE
    - RESOLVED
- Resolved 여부는 문제 해결 성공을 의미하지 않는다.

### 정리 및 확장 흐름
- 사용자는 하루 단위로 Session을 검토할 수 있다.
- 의미 있다고 판단한 Session만 정리 대상으로 선택한다.
- 블로그 초안 생성 여부는 사용자가 직접 선택한다.

## 결과
위 흐름을 통해 다음을 가능하게 한다.

- 하루 동안의 문제 해결 시도를 한눈에 파악할 수 있다.
- 의미 있는 기록만 선별해 자산화할 수 있다.
- 개발톡(DevTalk)의 기록이 문제와르르(ProblemWarruru)로 자연스럽게 연결된다.
